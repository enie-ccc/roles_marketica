2. Что будет после изменений

4. Кто к чему имеет доступ — по отделам

Обозначения

  • Полный — все данные, создание/редактирование/удаление
  • Да — есть доступ к странице
  • Отдел — данные своего отдела
  • Своё — свои записи и записи по своим проектам
  • — — нет доступа

4.1. Супер Админ

Роман, Сергей и другие. Полный доступ ко всему: все страницы, все данные, все действия, управление сотрудниками/ролями/правами/отделами.

Процесс согласования:

  1. Сотрудник создаёт заявку и назначает согласующих

  3. После одобрения всеми — бухгалтер получает уведомление

6. План реализации

Фаза 1 — Подготовка базы данных (1-2 дня)

  • Создаётся таблица «Отделы»
  • К сотрудникам добавляется привязка к отделу
  • Создаются новые роли, удаляются устаревшие
  • Пользователи автоматически переносятся на новые роли

Для пользователей ничего не меняется — система работает как раньше.

Фаза 2 — Обновление логики (3-5 дней)

  • Обновляются проверки прав
  • Добавляется фильтрация данных по отделам
  • Руководители начинают видеть данные отдела
  • Исправляются ошибки текущей системы

Фаза 3 — Обновление интерфейса (2-3 дня)

  • Обновляется боковое меню
  • В форме сотрудника — выбор отдела
  • Новая страница управления отделами
  • Куратор — галочка вместо роли

Фаза 4 — Тестирование (2-3 дня)

  • Проверка каждой роли
  • Telegram-уведомления
  • Экспорт и отчёты

Итого: ~2-3 недели. Все изменения обратимы, данные не удаляются.
